# 架构设计（iter-01）

## 领域划分

- 用户域：注册、登录、身份令牌
- 商品域：商品目录、商品列表查询
- 购物车域：购物项增删查
- 订单域：从购物车创建订单

## 边界与上下文

- 单体部署（Monolith API），按模块分层（handler/service/store）
- PostgreSQL 为主存储（用户、商品、购物车、订单）
- Redis 为读缓存（商品列表）
- NATS 为领域事件总线（订单创建事件）

## 事件流

1. 用户提交下单请求
2. 订单域读取购物车快照并持久化订单
3. 清空购物车
4. 发布 `orders.created` 事件到 NATS
5. 下游（后续迭代）可接入库存、营销、通知等消费者

## 一致性策略

- 订单写入与购物车清空：同一数据库事务内完成（本地强一致）
- 事件发布：当前为事务后发布（At-least-once 语义由消费者幂等保证）
- 商品列表缓存：Cache-Aside，失效后回源数据库

## 演进方向

- Outbox 模式确保 DB 与事件总线一致性
- CQRS 读写分离
- 服务拆分（用户、商品、订单）+ API Gateway
